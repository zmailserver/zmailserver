Setting Account Limits for Domains

You can limit the number of accounts that can be provisioned on a domain.  The maximum number of accounts allowed on a domain can be set when the domain is created.  You can also edit the domain settings to set this up or to change the configuration from the Domain Account Limits tab.

When the limit is reached,  if you  provision a new account, when you try to save the account, you see a message that the account limit has been reached and you cannot create the account.  Resource, domain wiki, spam and ham accounts are not counted against the limit you set.

  • To see a list of all accounts on a domain, select the domain from either the Navigation pane or the domain list in the Content pane and click View Accounts from the domain toolbar. The accounts are displayed in the Content pane.

Limiting COS Usage

When multiple Classes of Service (COS) are available,  you can select which classes of service can be configured and how many accounts on the domain can be assigned to the COS.

 In the Domain Account Limits tab, select the COSs that be can be assigned to accounts on the domain. Set the maximum number of accounts that can be assigned the COS. The limits for all COS cannot exceed the number set for the maximum accounts for the domain.

If COS usage for a domain is configured, when you create an account or edit an existing account, the Account Type section on the General Information page displays the COSs available for the domain.  

The number of account types assigned is tracked. You can see the number assigned/number remaining from any account's General Information tab.

To set or change the limit on a domain and set the COS limits

  1. Select the domain and open for edit.

  2. Open the Account Limits tab and enter the maximum number  of accounts for the domain in the Maximum accounts for this domain field.

  3. If you want to configure specific COS for the domain, click Add, otherwise go to Step 6.

  4. In the Add maximum accounts dialog Class of Service field

    • Enter the first letters of the COS and then select the COS from the list that displays.

    • In the Limit field, enter the number of accounts that can be created.

    • Click OK.

Note: The COS name that displays in the Account Type is taken from the COS Description field. If that field is blank, the COS name is displayed.

  1. Add additional COSs as described in step 4.

  2. Click Save. Click Close.

  3. Click the Browser Refresh button to update the information in the administration console.
