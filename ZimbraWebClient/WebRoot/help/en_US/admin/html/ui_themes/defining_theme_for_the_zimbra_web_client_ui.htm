Making UI Themes Available by COS

Zimbra Collaboration Server includes themes that can be used to customize the appearance of the Zimbra Web Client. The default user interface theme and the themes that users can select in the ZWC Preference>General page can be configured in the Class of Services or in individual accounts.

You can configure accounts to use only a specific theme, to be able to change to any of the themes that have been installed, or to be able to change to specific themes that are enabled.

Limit users to one theme

  1. Go to Configure>Class of Service and select the COS to edit.

  2. On the Themes page, select the theme that should be the Current UI theme.

  3. On the Features page, remove the checkmark from Change UI Themes. The UI theme is the theme listed in the Current UI theme field.

Let users access all of the installed Zimbra themes

  1. Go to Configure>Class of Service and select the COS to edit.

  2. Make sure that the Change UI Themes is checked on the Features page.

  3. On the Themes page, select all Themes available. Note: All themes in the list can be selected but the themes are not marked.

Let users access specific Zimbra themes

  1. Go to Configure>Class of Service and select the COS to edit.

  2. Make sure that the Change UI Themes is checked on the Features page.

  3. On the Themes page, enable Limit the Themes available to users.

  4. Check the box next to the themes you want to enable.

Save your changes.
