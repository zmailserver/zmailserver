Configure Message Retention and Disposal Policies

Retention and deletion policies can be configured as a global setting or you can configure COS-level policies instead of inheriting from the global settings.   

The policy sets the time threshold for items in the user's folders.  Users can select these policies to apply to their message folders in their account. They can also set up their own retention and deletion policies. Users enable a policy you set up or create their own policies from their folders’ Edit Properties dialog box.

Add Retention Policies

Note: To use the COS retention and disposal policies, make sure you check the box labeled Enable COS-level policies instead of inheriting from the policy defined in Global Settings on the Class of Service, Retention Policy page.

  1. Go to Configure > Global Settings, Retention Policy page.

  2. In the Retention Policies section, click Add.

  3. In the Add Retention Policy dialog box, type a descriptive policy name and set the retention range in  days, weeks, months, or years.

  4. Click OK.

The retention policy is not automatically enforced on a folder. If users delete an item in a folder that has not met the threshold of the retention policy, the following message is displayed, You are deleting a message that is within its folder’s retention period. Do you wish to delete the message?

Add Deletion Policies

  1. Go to Configure > Global Settings, Retention Policy page.

  2. In the Disposal Policies section, click Add.

  3. In the Add Retention Policy dialog box, type a descriptive policy name and set the disposal threshold  range in  days, weeks, months, or years.

  4. Click OK.

When the threshold for the deletion policy is reached, items are deleted from the account. They are not sent to the Trash folder. If the dumpster feature is enabled, they are sent to the dumpster, if it is not enabled, they are purged from the server.
