Managing Account Migrations from the Admin Console   bulk provisioning topic title

You can migrate accounts from an external directory, either Active Directory or an LDAP server. A migration wizard guides you through importing data from an external server. The wizard downloads account information from your directory and provisions the accounts on  Zimbra Collaboration Server.

When this is complete, the wizard generates a report in a .csv format with a list of new accounts.  

The wizard configuration details include:

  • Generating passwords for the accounts. You can configure the wizard to generate a random password for each account or use one password for all accounts and require users to change their passwords the first time they login. If you choose to generate random passwords for each account, make sure you download the list of provisioned accounts. The file is deleted when you close the wizard.

  • Setting up the LDAP directory connection. So that the Import Wizard can connect to the LDAP directory, information about the directory is entered. This includes the number of accounts to import, the server name, the base distinguished name (DN) of the server to be used as the root of the search, password, and a LDAP search filter to identify which mailboxes to import
