請求回執

備註：在 Zimbra Desktop 中，只有使用 Zimbra Web Client 時，才有「返回讀信回條」功能。

你可為你寫的郵件加個標記，要求收件人在打開你的郵件時發回條。 在你編寫郵件時，可在「寫信」工具列的選項選單中選擇返回讀信回條收件人可選擇返回、也可選擇不返回讀信回條。

在偏好設定郵件回條部分，用戶可為其標記回條請求的郵件設定進行偏好設定。可設定的選項有下列：

  • 不要傳送讀信回條。這是預設選項。即當收到一封附有回條請求的郵件時，該請求會被忽略。收件人不知道有該請求。

  • 永遠要求傳送讀信回條。當有「返回讀信回條」請求標誌的郵件被打開時，自動傳送一封讀信回條郵件。

  • 請詢問我。系統將詢問收件人是否要傳送讀信回條。

如果收件人選擇多封郵件，然後右鍵點擊並選擇註明已讀，則不會傳送讀信回條郵件。

 

備註：若傳送郵件給 ZWC 以外的其他系統上的用戶，則「返回讀信回條」標記不起作用。
