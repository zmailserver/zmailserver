Поддерживаемые условия и действия фильтров

Регистр букв правил фильтра не имеет значения, т.е. в правилах не учитываются заглавные буквы.

Условия

Поля сравнения включают следующие данные.

  • От. Используется для указания имени в поле «От:» заголовка сообщения электронной почты.

  • Кому. Используется аналогично параметру «От», но выполняется поиск указанных имен в заголовке «Кому» «Копия».

  • Копия. Используется аналогично параметру «От», но поиск выполняется в заголовке «Копия» «Копия».

  • Кому илиКопия. Используется аналогично параметру «От», но поиск выполняется в заголовках «Кому» и «Копия».

  • Тема. Поиск выполняется в заголовке сообщения «Тема».

  • Размер. Этот параметр используется для выбора сообщений, размер которых больше или меньше указанного. Размер сообщения включает любые файловые вложения. Этот параметр можно использовать для удаления сообщений слишком большого размера.

  • Дата. Этот параметр используется для указания сообщений, отправленных до или после определенной даты.

  • Тело письма. Параметры для поиска в теле письма: «содержит» или «не содержит». С их помощью можно указать слова для поиска в теле письма. Устанавливать фильтры для слов в файловых вложениях нельзя.

  • Подтверждение прочтения. Возможна фильтрация сообщений с подтверждением прочтения.

  • Вложение. Фильтрация сообщений, которые содержат или не содержат файловые вложения.

  • Адрес в. Проверка наличия адреса в контактах. В следующем поле можно указать, какой адрес.

  • Приглашение из ежедневника. Фильтрация сообщений, относящихся к ежедневнику.

  • Социальные сети. Фильтрация сообщений с сайтов социальных сетей.

  • Заголовок с именем. При выборе этого параметра появляется дополнительное поле ввода текста перед оператором сравнения (часть «содержит»). Этот параметр позволяет указывать любой заголовок сообщения. Можно указать не только стандартные поля «Кому:» или «От:», но также поля «Дата», «Обратный адрес» и другие специальные поля, которые могут содержаться в заголовке сообщения. Этот параметр можно использовать для фильтрации сообщений электронной почты с «неправильными» заголовками, т. е. заголовками, которые не содержат стандартную для электронных сообщений информацию. Иногда в спаме, который генерируется автоматически, отсутствует информация в таких полях, как «Отправитель» или «Обратный адрес». Для указания проверяемого заголовка используется второе текстовое поле (поле, которое находится справа от оператора сравнения).

Действия

  • Хранить в папке «Входящие». Сохраняет сообщения в папке «Входящие». Если почтовое сообщение не соответствует ни одному из правил фильтра, это действие выполняется по умолчанию.

  • Записать в папку. Перемещает сообщения в указанную папку.

  • Отменить. Удаляет сообщение, не доставляя его. Сообщения нет ни в одной папке, даже в Корзине.

  • Переслать. Сообщения перенаправляются на указанный адрес.

  • Пометить. Выберите вариант Прочитанное или С флагом.

  • Пометить меткой. Соответствующие сообщения можно пометить указанным тегом.

Несколько действий

Можно создать правило фильтра, состоящее из нескольких действий. Сочетания действий в правиле трактуются следующим образом.

  • Отменить. В сочетании с другими действиями этот параметр игнорируется, и выполняются другие действия.

  • Записать в папку. В результате нескольких действий «Записать в» несколько копий сообщения сохраняются в разных папках. Если указанная папка не существует, сообщение сохраняется в папке «Входящие».

  • Метка/Пометить. Эти действия применяются к сообщению, к которому применялось последнее предшествующее действие. В случае нескольких действий «Файл в» какие-то копии сообщений сохраняются с метками, какие-то — без.

  • Хранить в папке «Входящие». Можно определить несколько действий «сохранить», но при этом только одна копия сообщения будет сохраняться в папке «Входящие».

  • Переслать по адресу. Сообщения будут пересылаться по указанному адресу.
