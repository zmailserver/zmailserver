Использование адресных книг

Адресная книга  — это место для хранения контактных данных. Веб-клиент поддерживает два типа списка адресов.

•      Личные адресные книги, которые содержат поддерживаемые пользователем контакты. В адресной книге по умолчанию создаются следующие адресные книги: Контакты, Списки рассылки и Отправленные по электронной почте контакты. Можно создавать дополнительные адресные книги и предоставлять доступ к ним другим пользователям.

•      Пользователям в пределах организации доступна адресная книга Глобальный список адресов, иногда называемая «контактами компании». Глобальный список адресов может содержать удобные функции, например списки рассылки, облегчающие обмен информацией с целым отделом.

Примечание.   Записи в глобальном списке адресов изменять нельзя. Однако можно импортировать адрес, который содержится в отправленном или полученном сообщении электронной почты. Если имя находится в списке адресной книги, можно добавить дополнительные контактные данные.

Можно добавлять контакты в любую из адресных книг. Для создания контакта требуется только имя. Также можно создать подробную форму контакта, включающую полное имя, несколько адресов электронной почты, рабочий, домашний и другие адреса, номера телефонов и изображение. Также можно создавать списки групп контактов.

Адресная книга Отправленные по электронной почте контакты автоматически заполняется при отправке электронной почты на новый адрес, который отсутствует в других адресных книгах пользователя.

Администратор электронной почты может установить ограничение на максимальное количество записей, которые могут содержаться во всех адресных книгах пользователя. При достижении лимита исчезнет возможность добавлять контакты. Можно удалять контакты, чтобы освободить место.

При составлении сообщения электронной почты функция автозаполнения показывает список имен из адресных книг, соответствующих вводимому тексту.
