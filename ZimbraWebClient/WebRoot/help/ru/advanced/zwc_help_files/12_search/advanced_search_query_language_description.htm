Язык запроса расширенного поиска

Этот расшир раздел подробно описывает грамматику языка поиска, используемую @CollaborationProductShortName@ функцией поиска.

Структура языка поиска

Простой поиск можно выполнить, просто введя слово в поле поиска. Только слова (слова без оператора поиска) используются для поиска в операторе «content:», что соответствует любому тексту в сообщении.

Расширенный поиск можно выполнить, указав оператор поиска. Оператор поиска — это специальное ключевое слово, за которым следует двоеточие, а затем другие параметры, специфические для этого оператора. Например:

  •  В: входящие оператором является «в», а параметром — «входящие», в результате чего отображаются сообщения, которые содержатся в папке с именем «Входящие».

  •   от: имя_пользователя — оператором является «от», а параметром «имя_пользователя», в результате чего отображаются сообщения, которые содержат слово «имя_пользователя» в адресе электронной почты.

Перед любым ключевым словом можно поставить префикс «не», чтобы определить элементы, которые не содержат определенный критерий, например не в: входящие. В профиле поиска не учитывается регистр букв; это значит, что «в: входящие» равнозначно «в: Входящие». Знак минуса (-) является синонимом NOT . Поэтому не в: входящие равнозначно-в: входящие.

В большинстве случаев нет необходимости включать знаки пунктуации, строку поиска, поскольку они игнорируются кодом поиска. Но в определенных случаях это не так (например, поиск времени «9:30» в сообщении), и в этих случаях параметр поиска следует заключать в кавычки. Например: тема: 9.30 — будут найдены сообщения, в теме сообщения которых содержится время 9.30.

Допустимые знаки в параметре поиска.

  •  Следующие знаки можно использовать в параметре поиска только заключенными в кавычки: ~ ' ! # $ % ^ & * ( ) _ ? / { }[ ] ; :

  • Следующие знаки можно использовать в параметре поиска, только если они не являются первыми знаками в строке: - + < >

Примечание.   Специальные символы (например, товарные знаки, являющиеся частью слова), удаляются, когда слово индексируется для поиска. Они игнорируются при поиске. Например, поиск по слову Zimbra™ отображает результаты со словом Zimbra как с символом товарного знака, так и без него.

Несколько условий поиска

Если вводится несколько условий поиска (разделенных пробелами), по умолчанию они объединяются оператором «И». Если задать условие в: входящие Петр, будут отображаться сообщения, которые находятся в папке «Входящие» И содержат слово «Петр». Для поиска с использованием нескольких критериев, можно найти элементы соответствующие одному или всем указанным критериям. Со страницы расширенного поиска можно выполнить оба типа поиска.

Для всех областей поиска, кроме обычного поиска, правила следующие.

Поиск сообщений, соответствующих любым указанным критериям называется поиском с условием 'OR' , потому что, если сообщение содержит X или Y, оно считается соответствующим условиям поиска.

  • Поиск сообщений, которые содержат X и Y, называется поиском с условием И, поскольку сообщение должно соответствовать всем указанным критериям, чтобы считаться совпадением.

  • В запросе отображается только условие ИЛИ. Если задан параметр показывать запрос поиска на панели поиска по мере выбора в поле «Расширенный поиск», текстовое окно поиска будет обновляться и отображать полученный запрос. В профиле поиска типа AND  слово AND не отображается.

Совет. Использование скобок с условиями И и ИЛИ. Слова в скобках считаются единицей. Например, from: (Иван Василий) равнозначно условию от: Иван И от: Василий. Если использовать ИЛИ в скобках, от: (Иван или Сидоров), результат поиска будет от: Иван ИЛИ от: Сидоров.

Использование * (звездочки) в качестве подстановочного знака в профиле поиска

Звездочку (*) можно использовать в качестве подстановочного знака в конце слова при поиске содержимого, которое включает слова с похожим написанием.

Используйте звездочку *  как подстановочный знак. Например, строка поиска ты*  выдает элементы ты, тыква, тысяча и т.д.

Примечание.   Двухбайтовый символ звездочки в запросе поиска не рассматривается как подстановочный знак и игнорируется при поиске.
