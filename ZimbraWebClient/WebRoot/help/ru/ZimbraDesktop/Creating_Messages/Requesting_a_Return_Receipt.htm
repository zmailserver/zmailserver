Запрос подтверждения прочтения письма

Примечание. В Zimbra Desktop функция подтверждения прочтения письма доступна только в случае использования Zimbra Web Client .

Для создаваемого сообщения можно установить флаг, указывающий на то, что ожидается подтверждение прочтения письма после того, как получатель его откроет.  При создании сообщения следует выбрать пункт Запросить подтверждение прочтения в меню Параметры на панели инструментов «Написать письмо». Получатель может не согласиться на отправку вам подтверждения прочтения письма.

В разделе Подтверждение прочтения на вкладке «Настройки > Почта» пользователи могут устанавливать свои настройки для сообщений с запросами на подтверждение прочтения. Возможны следующие варианты.

  • Никогда не отправлять подтверждение прочтения. Это настройка по умолчанию. При получении сообщения с запросом подтверждения прочтения этот запрос игнорируется. Получатель не знает о запросе.

  • Всегда отправлять подтверждение прочтения. Подтверждение прочтения отправляется автоматически при открытии сообщения с запросом подтверждения прочтения.

  • Спрашивать меня. Получателю предлагается выбрать, нужно ли отправлять подтверждение прочтения.

Подтверждение прочтения не отправляется, если получатель выбрал несколько сообщений, щелкнул их правой кнопкой мыши и выбрал Пометить как прочитанное.

 

Примечание. Флаг подтверждения прочтения действует только для сообщений, отправленных пользователям системы ZWC
