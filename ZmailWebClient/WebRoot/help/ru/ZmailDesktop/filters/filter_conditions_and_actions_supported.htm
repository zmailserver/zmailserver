Поддерживаемые условия и действия фильтров

Регистр букв правил фильтра не имеет значения, т.е. в правилах не учитываются заглавные буквы.

Условия

Поля сравнения включают следующие данные.

  • От. Используется для указания имени в поле «От:» заголовка сообщения электронной почты.

  • Кому. Используется аналогично параметру «От», но выполняется поиск указанных имен в заголовке «Кому» .

  • Копия. Используется аналогично параметру «От», но поиск выполняется в заголовке «Копия».

  • Кому или Копия. Используется аналогично параметру «От», но поиск выполняется в заголовках «Кому» и «Копия».

  • Тема. Указываются данные из заголовка «Тема».

  • Именованный заголовок. При выборе этого параметра появляется дополнительное поле ввода текста перед оператором сравнения (часть «содержит»). Этот параметр позволяет указывать любой заголовок сообщения. Можно указать не только стандартные поля «Кому:» или «От:», но также поля «Дата», «Обратный адрес» и другие специальные поля, которые могут содержаться в заголовке сообщения. Этот параметр можно использовать для фильтрации сообщений с «неправильными» заголовками, т. е. заголовками, которые не содержат стандартную для электронных сообщений информацию. Иногда в спаме, который генерируется автоматически, отсутствует информация в таких полях, как «Отправитель» или «Получатель». Для указания проверяемого заголовка используется второе текстовое поле (поле, которое находится справа от оператора сравнения).

  • Размер. Этот параметр используется для указания сообщения, размер которого больше или меньше указанного. Размер сообщения включает любые файловые вложения. Этот параметр можно использовать для удаления сообщений слишком большого размера.

  • Дата. Этот параметр используется для указания сообщения, отправленного до или после определенной даты.

  • Тело письма. Параметры для тела письма «содержит» или «не содержит» позволяют указать соответствующие слова в теле письма. Устанавливать фильтры для слов в файловых вложениях нельзя.

  • Подтверждение прочтения.Возможна фильтрация сообщений с подтверждением прочтения.

  • Вложение. Можно установить фильтр для сообщений, которые содержат или не содержат файловые вложения.

  • Адрес в. Проверка наличия адреса в контактах. В следующем поле можно указать, какой адрес.

  • Приглашение из ежедневника. Возможна фильтрация сообщений, имеющих отношение к ежедневнику.

Операторы сравнения включают следующие данные.

  • Точно соответствует/не точно соответствует. Указывает точное соответствие. Например, если в поле Тема точно соответствует указать Яблоко, то соответствием будет «Яблоко», но не «Яблоки» или «Грузовик с яблоками».

  • Содержит/не содержит. Этот параметр указывает, что строка темы должна содержать указанную подстроку. Например, если указать Тема содержит «яблоко», то соответствием может быть «Я хочу яблоко».

  • Соответствует/не соответствует условию поиска по шаблону. Подстановочный знак звездочка (*) используется в поле сравнения почтового фильтра для подстановки одного или более знаков фильтра. Например, если указать, что тема должна соответствовать слову «яблоко*», то соответствиями будут «яблоки», «яблоки-пироги», но не «мои яблоки».

Действия

  • Хранить в папке «Входящие». Сохраняет сообщения в папке «Входящие». Если почтовое сообщение не соответствует ни одному из правил фильтра, это действие выполняется по умолчанию.

  • Записать в папку. Перемещает сообщения в указанную папку.

  • Отклонить. Удаляет сообщение, не доставляя его. Сообщения нет ни в одной папке, даже в Корзине.

  • Переслать. Сообщения перенаправляются на указанный адрес.

  • Пометить. Выберите вариант Прочитанное или С флагом.

  • Пометить тегом. Соответствующие сообщения можно пометить указанным тегом.

Несколько действий

Можно создать правило фильтра, состоящее из нескольких действий. Сочетания действий в правиле трактуются следующим образом.

  • Отклонить. В сочетании с другими действиями этот параметр игнорируется, выполняются другие действия.

  • Записать в папку. В результате нескольких действий «файл в» несколько копий сообщения сохраняются в разных папках. Если указанная папка не существует, сообщение сохраняется в папке «Входящие».

  • Тег/пометить. Эти действия применяются к сообщению, к которому применялось последнее действие. В случае нескольких действий «файл в» какие-то копии сообщений сохраняются с тегами, какие-то — без.

  • Хранить в папке «Входящие». Можно определить несколько действий «сохранить», но при этом только одна копия сообщения будет сохраняться в папке «Входящие».

  • Переслать по адресу. Сообщения будут пересылаться по указанному адресу.
