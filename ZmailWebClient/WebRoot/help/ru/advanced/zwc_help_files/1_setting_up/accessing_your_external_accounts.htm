Доступ к внешним учетным записям

Вместо того, чтобы входить в каждую внешнюю учетную запись, почту из своих почтовых ящиков можно получать непосредственно с помощью учетной записи @ClientNameShort@. При настройке доступа к другим внешним почтовым ящикам можно указать, где будут храниться сообщения, и создать образ, используемый для ответа или пересылки сообщений, получаемых с этой внешней учетной записи.

Сообщения из внешней учетной записи не поступают автоматически в учетную запись @ClientNameShort@. Щелкните значок Обновить на панели инструментов, чтобы получить новые сообщения.

Если настраивается дополнительная внешняя учетная запись POP, можно указать параметры доступа по протоколу POP при загрузке сообщений в ваш почтовый ящик.

Добавление внешних учетных записей

  1. Перейдите на страницу Настройки>Учетные записи.

  2. Нажмите кнопку Добавить внешнюю учетную запись. Имя учетной записи Новая внешняя учетная запись отображается в столбце «Имя учетной записи» и текстовом поле Настройки внешней учетной записи>Имя учетной записи. Этот параметр можно изменить.

  3. В разделе Настройки внешней учетной записи введите адрес электронной почты внешней учетной записи.

  4. В текстовом поле Имя учетной записи введите описательное слово для определения этой учетной записи. Оно используется для обозначения внешней учетной записи.

  5. Выберите тип учетной записи: POP или IMAP.

  6. В текстовом поле Имя пользователя учетной записи введите имя, связанное с данной учетной записью электронной почты. Иногда имя пользователя является частью электронного адреса до знака @, иногда это полный электронный адрес пользователя.

  7. В текстовом поле Почтовый сервер введите имя сервера для внешней учетной записи.

  8. В текстовом поле Пароль введите пароль, используемый для входа в данную внешнюю учетную запись. Чтобы проверить правильность введенного пароля, установите флажок Показать пароль.

  9. Если ваш провайдер использует порт, отличный от порта по умолчанию, измените его в области Дополнительные настройки. В случае использования безопасного подключения включите параметр Во время доступа к серверу используется зашифрованное подключение (SSL).

    10. Провайдер должен сообщить вам, если не используется стандартный порт IMAP (143) или POP (110).

  10. Нажмите кнопку Проверить настройки. Информация будет проверена.

  11. Выберите папку для загрузки сообщений.

  12. Чтобы сообщения, полученные с помощью вашей учетной записи, затем удалялись из внешней учетной записи, установите флажок Удалять сообщения с сервера после загрузки.

  13. Укажите имя в поле «От», адрес для ответа и настройки подписи для исходящих сообщений.

  • Введите имя, которое должно быть указано в поле От исходящих сообщений (например, Иван). Это имя, которое будет отображаться перед вашим адресом электронной почты.

  • Чтобы направлять ответы на сообщения электронной почты от этого образа на имя и адрес, отличные от указанных в поле От, выберите параметр Указывать в поле «Обратный адрес» сообщений.  Введите имя в текстовом поле.

  • Чтобы связать подпись с образом, щелкните Подпись и добавьте новую подпись.

  1. Нажмите кнопку Сохранить.
