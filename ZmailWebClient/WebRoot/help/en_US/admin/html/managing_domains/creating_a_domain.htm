Adding a New Domain

One domain is identified during the installation process and additional domains can be added. When you add a domain you configure the Global Address List (GAL) mode and the Authentication mode for the domain. You can also set up virtual hosts, and limit the number of accounts that can be created on the domain.

To add a domain, click Configure>Domains in the Navigation pane and in the gear icon and select New. The Create New Domain wizard walks you through the steps to add the new domain.

  1. On the General Information dialog box, enter the following information.

    • Domain name. Enter the full name of the domain ( domainexample.com).

    • Public service host name. Enter the host name of the REST URL. This is the URL that is used when sharing email folders, Briefcase folders, sharing Task lists, Address Books, and Calendars. Setting this up includes configuring the Public Service Protocol and the Public Service Port.

    • Inbound SMTP host name. If your MX records point to a spam-relay or any other external non-Zimbra server, enter the name of the server here

    • Description. Text field that displays in the Domain list in the Content pane.

    • Maximum accounts for this domain. Enter the maximum number of accounts that can be created on this domain.  The number entered here is displayed in the Account Limits tab.  The default is no limit (blank field).

    • DefaultClass of Service (COS) for the domain. If you do not enter a COS, the default COS is used. The COS entered here is  automatically assigned to accounts created on the domain if no other COS is selected.

    • Domain status. The domain status is active in the normal state. You may want to change this status when you setting up the domain if you are not ready to use the domain.

    • Notes you want to add about the domain.

  1. GAL Search Setting. Choose the GAL mode, either Internal, External, or Both. The default is Internal.

    • (Optional) Change the maximum number of search results returned for a GAL Search.  The default is 100 results.

    • Enter the GAL Sync Account name. This account syncs to the LDAP, and all GAL contacts in LDAP are added to the Address Book.

    • Enter the data source name for the GAL sync.

    • If you selected an external GAL, configure the external LDAP settings.

    • Set  the Internal GAL polling interval in days, hours, minutes, or seconds. This is the frequency that the GAL sync account is updated.

  1. Configure the Authentication Mode.

    • If you selected Internal as the GAL mode, no other authentication configuration is required. Click Next.

    • If the authentication mechanism is External LDAP or External Active Directory, the next step is to configure the authentication mode .

      1. Select either External LDAP or External Active Directory.

      2. Enter the external server name

      3. For External Active Directory, enter the active directory domain name. The external server port is 389.

      4. If you will use SSL, check the box to Use SSL to connect to the server.

      5. For external LDAP, enter the LDAP Bind Template name.

      6. The Authentication Settings summary page is displayed with the information you just entered. Enter a username and password to test the authentication settings. Click Test.

      7. The Authentication Test Results page displays the test results. If the results failed, click Previous, to resolve the issues described in the Details text box. Click Finish, if the test past.

       

       

  1. Virtual Hosts. (Optional) Enter the virtual host address. Virtual hosting allows you to host more than one domain name on a server. When you create a virtual host, this becomes the default domain user login.

  2. Briefcase. Configure the name that should be displayed in the prompt on the Authentication Required dialog used to log into Briefcase folders shared with external guests. The default is Zimbra.

  3. Click Finish to create the domain.
