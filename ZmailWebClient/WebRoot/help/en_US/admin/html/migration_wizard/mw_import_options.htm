Import Options

When you import account files with the Migration Wizard, only email messages, calendar and contacts are imported. Other types, such as meeting requests, notes/journal, tasks, rules and alerts, personal distribution lists, and files are not imported.

The Import Options dialog is used to set rules about what files should be imported.

  1. The checkboxes for Import Junk-Mail Folder and Import Deleted Items Folder determine whether or not to include mail currently in those folders in Outlook.

  2. The check box for Ignore previously imported items can be used in cases where an earlier import operation was interrupted. If checked, the import process picks up where it left off, rather than importing duplicates.

  3. Check the Date Filter check box to import mail only after a certain date. Enter the date. Messages before that date are not imported. This option lets you discard older messages.

  4. In the Number of Mailboxes to import simultaneously field, set the import concurrence level. Since the import is likely to happen while both Exchange and Zimbra servers continue to service other mail accounts, you should use care when increasing this number, as it may affect server performance.

  5. For Domino migration, check Old-email ID, if the old email address of the account should be replaced by the new Zimbra email address in all the meetings organized by that user

For Exchange migration, Two Calendar options can be set. Check the first box, if the old email address of the user should be replaced by the new email address in all the meetings organized by the user.  Check the second box, if users' private appointments should be migrated. These appointments will not be marked as private on the Zimbra server.

  1. Click Next.  The Begin Import Process? Message box is displayed. Click OK to begin importing.

 

  • Important: Users can continue to work in their mailbox while their mail is being imported, although any operation they do after that mail or folder has been imported are not be picked up on the Zimbra side. To ensure that the imported mailbox is consistent with the old mailbox, users should be informed that their mailboxes is not available over some scheduled downtime period.
