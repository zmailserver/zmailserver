Domino Source and Destination Mailboxes

In this dialog, list the users who should be migrated to the Zimbra mailbox server.

You can select users with either of the following applications:

  • Object Picker, to select individual accounts to migrate to Zimbra.

  • Query Builder, to select groups of accounts to migrate to Zimbra.

When importing accounts, the migration wizard assumes that:

  • Imports are by a single domain. For example, you might import users from marketing.domain.com on your Domino server to zimbra.domain.com on your Zimbra server.

  • The userâ€™s Common Name ( CN) in Domino will be the first portion of the email address on the Zimbra server. To continue the above example, a user who has an account for joe@marketing.zimbra.com on Domino would become joe@marketing.zimbra.example.com, on your Zimbra server.

 

Click Object Picker to select specific accounts to migrate. The Object Picker lists all the users in the Domino name and address book (NAB).

    1. Select the users that should be migrated to ZCS. To limit the number of users to migrate at one time, choose specific email addresses . You can include a substring. For example, you can specify to migrate all email addresses with the first letter of A, or adam@*.com, or adam@acme.com.

    2. You can continue to use the Object Picker to add more names.  When complete, click OK.

    3. When all the names are displayed in the Source and Destination Mailboxes dialog, click Next.

 

Click Query Builder to select groups of accounts to migrate.

    1.  Enter the search base for the LDAP query

    2. Construct the filter to identify which mailboxes to import.

    3. Check Subtree to search the entire subtree under the Base DN

    4. Click OK.

The Wizard now checks to see if the accounts selected exists on the Zimbra server.
