Using the Zimbra Migration Wizard for Microsoft Exchange

The Zimbra Migration Wizard is used to move MicrosoftÂ® Exchange server email accounts to the Zimbra server and to import the email, calendars and contact information for selected Exchange users. The import process replicates the user's Outlook folder hierarchy, imports email messages, calendar and contacts on the Exchange server. Keywords are converted to tags in Zimbra.

IMPORTANT: The Migration Wizard utilizes the Exchange MAPI provider. You must run the wizard from a Microsoft Windows computer that has either the Outlook or the Exchange Administration tools installed.

  • You must have full administrator privileges on the mailboxes to be imported

  • You must create a temporary MAPI profile to conduct the migration.

  • The account under which you log into Windows must have permission to open the temporary MAPI profile

  • This account must be a member of the Windows domain administrators' group

  • The selected MAPI profile specifies a single account that will be used to open and import all mailboxes

Read the Migration Wizard Guide for Exchange for more information. You can find this on the administration console Help Desk page.

Before you begin

The Migration Wizard asks questions in a series of dialog boxes about how to set up the new accounts.

Before you start, you should:

  • Determine which domain users' accounts should migrate to. You can import to one domain at a time.

  • Decide whether to migrate individual accounts or large groups of accounts at one time.

  • Change the mail relay information to point to the Zimbra server instead of the Exchange server. This will prevent any lost mail while the files are being imported.

The user's Common Name ( CN) in Exchange is the first portion of the email address on the Zimbra server.

You can specify if email in Junk folders or Deleted Items folder is imported, and you set a latest receive date for old email to be imported.

By default, messages larger than 10 MB will not be imported. The message size includes the message and attachments. If the message is larger than 10 MB, a warning is logged to Migration Wizard log file, which can be reviewed from the Import Complete dialog when the migration is complete.

How to Migrate Accounts

  1. Click the Migration Wizard icon on the administration console to download executable to your computer.

  2. Go to the directory where the file is saved. Click the Migration Wizard.exe, to open the wizard. The Welcome dialog appears.

  3. Follow the dialogs. Below is the dialog sequence.

  • Import Destination. Enter the host name and administration user name and password.

  • Destination Domain. Select the Zimbra domain name.

  • MAPI Log on. Select the MAPI profile you created for use with the Migration Wizard.

  • Source and Destination Mailboxes. Define which users should be migrated.

  • Target Account Verification. Displays the number of accounts that exist and the number of accounts that do not exist on the Zimbra server.

  • Account Provisioning. All accounts that do not exist are shown in the Unprovisioned Accounts list box.

  • Import Options. Configure whether to import junk, deleted mail,  and other older mail and set the number of mailboxes to import simultaneously.

  • Import In Progress. When the import is complete, errors or warnings are displayed.
