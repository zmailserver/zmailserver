設定 S/MIME 以傳送及接收已簽署及加密電郵訊息

備註： 這是 ZCS Network Edition 的專門功能，並只可用於 Advanced Web Client。

如果你的帳戶已啟用 S/MIME 功能，你可以傳送及接收電子簽署及已加密的郵件。

若需與 @ClientNameLong@ 使用 S/MIME，你必須有一個適合 S/MIME 簽署及加密的安全電郵簽署憑證。附有私人金鑰的憑證必須在 Windows 電腦及 Mac OSX 電腦的本地憑證存放區，或瀏覽器的憑證存放區（如使用 Firefox）安裝。如需瞭解怎樣安裝你的憑證的相關資訊，請參閱適當的電腦或瀏覽器的說明書。

若要收發已簽署及加密的郵件，你的公用鑰匙及收件人的公用鑰匙必須互相交換。若要取得收件人公用鑰匙的副本，你可利用你的憑證傳送電子簽署的郵件給收件人，並請求對方在回覆中傳送電子簽署的郵件給你。當接收到郵件，收件人的憑證會被儲存到通訊錄中收件人的通訊錄頁面。

請注意，在處理已加密郵件時留意以下的差別：

  • 如果你正分享你的收件箱，受委派的用戶無法在他們的電腦讀取你已加密的郵件。需要你的私人金鑰以讀取已加密的郵件。

  • 你不能儲存已加密郵件的草稿。

  • 你不能執行在已加密郵件中的拼字檢查。

  • 你無法在已加密郵件的正文或附件中搜尋文字。  只有標題資訊如發送到或寄件人、郵件傳送日期或主題可在搜尋中找到。
