閲覧確認を要求する

備考：Zimbra Desktopでは、Zimbra Web Clientを使用しているときのみ、閲覧確認を要求できます。

作成するメッセージには、受信者がそのメッセージを開いたときに閲覧確認メッセージを返信するようにフラグを付けることができます。 メッセージを作成するときに、作成ツールバーの[オプション]メニューの[閲覧確認を要求]を選択します。受信者は閲覧確認を返信するかどうかを選択できます。

[プリファレンス]>[メール]の[閲覧確認]領域で、閲覧確認要求のフラグを付けるメッセージに対するプリファレンスを設定できます。以下のオプションを設定できます。

  • 閲覧確認を送信しない。これはデフォルトです。閲覧確認の返信要求が付いているメッセージを受信しても、その要求は無視されます。受信者は要求に気付きません。

  • 常に閲覧確認を送信する。閲覧確認の返信要求が付いているメッセージを開くと、閲覧確認メッセージが自動的に送信されます。

  • 確認する。 閲覧確認を送信するかどうかを受信者に確認します。

受信者が複数のメッセージを選択し、右クリックして[既読にする]を選択した場合は、閲覧確認メッセージは送信されません。

 

備考：閲覧確認の要求フラグは、ZWC以外のシステムを使用するユーザーに送信したメッセージでは機能しません。
